UML Diagrams

There are two main categories and 14 different types of UML diagrams, each of which is used in a different situation.

Structure diagrams: show the static relationships between the components in the system.
Behavior diagrams: show how the components in the system react to each other, they capture how the system changes, and in some diagrams, how it changes over time.

Examples of Structure diagrams:

  • Package Diagram: Type of diagrams that simplifies complex class diagram by grouping classes into packages
  • Class Diagram: An object or a set of objects that share a common structure and behavior
  • Deployment Diagram: A type of UML diagrams that models the physical deployment of artifacts on nodes
  • Object Diagram: A static object diagram is an instance of a class diagram; it shows a snapshot of the detailed state of a system at a point in time.
  • Component Diagram: A diagram visualizing the organization of system components and the dependency relationships between them
  • Composite Structure Diagram: A UML diagram showing the internal parts of a class

    • Examples of Behavior diagrams:

  • Activity Diagram: Show the workflow from a start point to the finish point detailing the many decision paths that exist in the progression of events contained in the activity.
  • Use Case Diagram: A graphical depiction of a user's possible interactions with a system
  • State Diagram: Diagram that depicts the state that a system or component can assume, and shows the events or circumstances that cause or result from a change from one state to another
  • Interaction Diagram: A diagram envisioning the interactive (dynamic) behavior of a system

    • Examples of Interactional diagrams:

  • Sequence Diagram: A type of interaction diagram because it describes how—and in what order—a group of objects works together.
  • Timing Diagram: A behavioral modeling diagram focusing on timing constraits
  • Communication Diagram: A UML chart that represents the flow of messages in a system
  • Interaction Diagram: A form of activity diagram in which the nodes represent interaction diagrams
